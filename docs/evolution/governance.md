# Evolution Engine — Governance Model

## Approval Matrix

| Blast Radius | Auto-Approve Threshold | Requires | Escalation |
|---|---|---|---|
| `SingleFile` | score ≥ 0.70 | Passing experiment (Shadow or Canary) | Admin if confidence < 0.80 |
| `SingleRepo` | score ≥ 0.80 | Canary experiment + PR review | Admin if pValue > 0.05 |
| `MultiRepo` | Never auto | Human Reviewer approval + PR | Admin mandatory |
| `Ecosystem` | Never auto | Admin board approval + PR | Board vote required |

Score range: [0, 1]. Proposals below 0.60 are automatically archived as `Rejected`.

## Roles

### Autonomous (AI Engine)
- Generates proposals from evaluated opportunities
- Runs Shadow and Canary experiments within budget limits
- Auto-approves `SingleFile` proposals meeting threshold
- Records every action to AuditTrail
- Cannot approve `MultiRepo` or `Ecosystem` proposals

### Reviewer (Human)
- Reviews PRs generated by `SingleRepo` proposals
- Can approve or reject via GitHub PR review
- Can trigger manual re-evaluation via `evolution-cli reassess <id>`
- Receives webhook notifications on `PendingApproval` status

### Admin (Board)
- Final escalation authority for `MultiRepo` and `Ecosystem` proposals
- Can activate/deactivate KillSwitch
- Can override rate limits for urgent optimisations
- Reviews weekly ROI scoreboard; adjusts budget limits in `platform_config.json`

## Escalation Paths

### Level 1 — Autonomous gate failed
- Confidence below 0.80 or pValue above 0.05 after Canary
- Engine sets status `PendingApproval`, notifies Reviewer via webhook
- SLA: 48 hours before proposal expires and is archived as `Rejected`

### Level 2 — Reviewer escalates to Admin
- Proposal touches more than one repository
- Blast radius upgraded during experiment (unexpected scope expansion)
- Reviewer sets `requires_admin: true` on GitHub PR; Admin notified via mention

### Level 3 — Emergency halt
- Admin activates KillSwitch via `evolution-cli kill-switch enable --reason "<text>"`
- All running experiments paused; pending proposals frozen
- Resume requires explicit `evolution-cli kill-switch disable` with justification in AuditTrail
