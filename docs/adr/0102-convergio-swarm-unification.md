# ADR 0102: Convergio Swarm Unification

Status: Accepted | Date: 21 Mar 2026 | Plan: Phase 1

## Context

Convergio had fragmented agent infrastructure: file-based bus, duplicate agent definitions (.claude/ vs .github/), global config pollution, no multi-tool support. The daemon already had a complete IPC engine but shell scripts bypassed it.

## Decision

1. **Daemon is single source of truth** — all agent registration, messaging, telemetry go through daemon API on :8420. No file-based bus.
2. **Global overlay via toggle** — `convergioOn/Off` creates/removes symlinks in `~/.claude/` (commands, agents, rules, reference). Works in ANY repo.
3. **Canonical agent format** — ONE source in `claude-config/agents/`. Tool-specific formats generated by `convergio sync-agents`.
4. **Multi-tool launcher** — `convergio <agent> [name] [--tool claude|copilot|opencode]` launches any tool with any model.
5. **Named sessions** — agents register with custom names in daemon IPC, enabling cross-terminal messaging.
6. **Calibration wired** — `calibrate-estimates` + `plan-post-mortem` mandatory after plan closure.

## Consequences

- Positive: tool/model/machine agnostic swarm, no config pollution, centralized telemetry
- Negative: requires daemon running for full functionality (graceful degradation when offline)

## Enforcement

- Rule: `rules/enforcement.md` updated with calibration + post-mortem steps
- Check: `convergio status` shows daemon + symlink state
- Ref: ADR-0002 (agent-bridge-decoupling)
